# X 写作风格手册

这个文件用于逐步沉淀账号自己的写作风格。每写完一篇文章，都可以回头更新一次：保留有效表达，删掉空泛句式。

## 账号定位

核心方向：

- AI 编程
- AI 创作自媒体
- 自动化工作流
- AI 工具实战
- 一人完成原本需要团队协作的交付
- AI 视频生成工具实测
- AI Coding 工具对比

一句话定位：

> 用 AI 写代码、做内容、搭自动化，把想法变成能运行、能发布、能复用的工作流。

账号长期主线：

> 实测 AI 工具，拆解 AI 工作流，把想法变成能跑的作品。

## 核心气质

要像一个亲自用过、踩过坑、会拆流程的人。

不是资讯搬运号。
不是参数党。
不是单纯提示词号。
不是“无脑赚钱”号。

文章要有这几种感觉：

- 直接：开头就给结论，不绕。
- 实战：每个观点都能落到操作步骤。
- 克制：不夸大收益，不喊口号。
- 有判断：敢说“新手最容易错在哪里”。
- 有边界：说明适合什么、不适合什么。

当前最适合强化的识别度：

- 实测：不是看发布会，而是自己跑一遍。
- 对比：同一 Prompt、同一任务、不同工具。
- 工作流：把单点工具串成可复用流程。
- 作品：最后要有截图、视频、网页、脚本或文件。

## 常用表达结构

### 1. 反常识开头

适合第一条 Hook：

- 如果你只把 X 当成 Y，基本只用了它很小一部分能力。
- 刚装 X，先别急着研究每个按钮。
- 新手最容易犯的错，不是不会用，而是上来就许愿。
- X 真正值钱的地方，不是 A，而是 B。

### 2. 重新定义

这是当前文章里最稳定的风格：

- Codex 不是聊天框，而是 AI 执行环境。
- Codex 不是许愿机，而是需要你会管理的 AI 同事。
- AI 不负责替你思考，它负责加速你的流程。
- 真正有价值的不是会用工具，而是把重复流程沉淀成资产。

### 3. 五层/四步/三类拆解

适合教程型内容：

- 新手路线分 5 层。
- 给 Codex 任务时固定 4 件事。
- 浏览器能力分 3 类。
- 每篇 X 文章按 Hook、Body、TL;DR、CTA 来写。

### 4. 工作流句式

适合把复杂能力讲清楚：

```text
读项目 -> 做计划 -> 小步修改 -> 跑测试 -> 看浏览器 -> Review diff -> 留评论 -> 再修一轮 -> 沉淀规则
```

```text
输入素材 -> 提炼观点 -> 重组结构 -> 生成初稿 -> 配图 -> 合规检查 -> 发布复盘
```

去 AI 味工作流：

```text
检测 AI 痕迹 -> 删套话 -> 改成人话 -> 补真实细节 -> 做审美检查 -> 沉淀个人风格
```

学习系统工作流：

```text
目标 -> 知识地图 -> 每日任务 -> 主动练习 -> 错误记录 -> 间隔复测 -> 项目交付
```

### 5. 实测对比句式

适合工具评测：

- 相同 Prompt 下，A 和 B 到底差多少？
- 我用同一个任务测了 A / B / C，结论和我预期不一样。
- 这个工具不是不能用，而是适用场景很窄。
- A 适合快速出样，B 适合最终成片。
- 不要只看画质，还要看成本、速度、稳定性和可控性。

### 6. 品牌案例句式

适合“想法 -> 作品”：

- 我把一个想法交给 AI，最后做成了一个能运行的作品。
- 这不是单个 Prompt，而是一条完整生产链路。
- 最有价值的不是生成结果，而是这套流程下次还能复用。

### 7. 去 AI 味句式

适合写作方法论：

- 去 AI 味，不是把话写口语一点。
- 真正的 AI 味，是看起来都对，但没有判断、细节和现场感。
- 改文章不要一上来润色，先定位哪里空、哪里套、哪里像模板。
- 人味不是语气松散，而是有经历、有边界、有取舍。
- 最差的去 AI 味，是把所有文章改成另一个“人味模板”。

### 8. 学习系统句式

适合 AI 学习 / Codex 学习仓库：

- AI 学习最大的坑，是把“拿到答案”误认为“获得能力”。
- 答案不等于能力，能完成任务才算真的学会。
- 不要让 AI 只给你学习计划，要让它维护学习状态。
- 把学习做成本地仓库，Codex 才能从真实进度继续推进。
- 没有练习、错误和复测，学习计划只是漂亮目录。

## 文章结构

优先用 8-10 条 Thread。

### 第 1 条：Hook

目标：3 秒内让人知道“这篇和我有关”。

写法：

- 先给反常识判断。
- 不要铺背景。
- 不要上来感谢、寒暄、堆 hashtag。

### 第 2-3 条：问题和框架

目标：让读者知道为什么要继续看。

写法：

- 说清楚新手常见误区。
- 给出这篇文章的总框架。

### 中间条：每条只讲一个点

目标：让读者可以滑着读完。

写法：

- 每条 80-160 字左右。
- 每条只解释一个判断。
- 能列清单就列清单。
- 每 2-3 条配一张图。

### 最后一条：TL;DR + CTA

目标：帮助收藏和互动。

写法：

- 用 4-5 个 bullet 总结。
- CTA 以评论问题为主，不要硬广。

常用 CTA：

- 你还想看哪个工具对比？
- 评论区说一个你想自动化的重复工作。
- 你更想看完整 Prompt，还是完整工作流？
- 如果你也在测这个工具，欢迎把结果贴出来对比。

## 标题风格

标题要短、具体、有判断。

优先：

- 刚装 Codex 桌面版，别先学按钮
- 如果你只拿 Codex 聊天，真的浪费了
- Codex 不是聊天框，而是 AI 执行环境
- 相同 Prompt 下，Seedance mini 和 4k 差多少？
- 我用 Codex 跑完了一条完整内容生产工作流
- 从一句想法到可运行作品，我让 AI 跑完整链路

避免：

- Codex 完整教程大全
- 一文讲透 Codex 的所有功能
- 2026 最新最全无敌教程

## 视觉风格

配图以“大字 + 清晰结构”为主。

要求：

- 字少，手机缩略图能一眼看清。
- 主标题 4-8 个字优先。
- 暗色科技底 + 高对比大字可作为长期风格。
- 图里不要塞太多说明文字。
- 图文作用分开：图抓注意力，正文讲逻辑。

当前可延续的视觉关键词：

- 黑色 / 深石墨
- 青绿高亮
- 橙色强调
- 工作流箭头
- 大标题居中
- 简洁卡片和流程节点

新增可参考的高冲击封面风格：

- 参考图：`references/visual-style/xiaohongshu-cover-high-impact-reference.png`
- 用途：标题图、封面图、强 Hook 图，尤其适合方法论、Skill、工作流、反直觉观点、自媒体封面分析。
- 视觉语言：白色或浅灰纸感底，红黑白高对比，超大中文主标题，关键词红黑交替，右侧放立体卡片/手机界面/成果预览。
- 可用元素：红色笔刷、喷溅、下划线、准星、星芒、鼠标指针、靶心、箭头、角标、底部标签。
- 构图原则：左侧 45%-60% 放巨大标题，右侧放成果预览，底部放“附 Prompt / Skill / 模板 / 工作流”等钩子标签。
- 边界：参考风格，不复制原图内容和具体素材；正文解释图不必全部高噪音化，优先保证主标题在手机缩略图里清晰可读。

### 封面锚点系统

已学习 `ponyodong2026/ponyo-cover-anchor-system` 的方法，后续标题图默认先过“信息密度 × 视觉锚点”检查。

本地学习笔记：`references/visual-style/ponyo-cover-anchor-system-notes.md`

核心规则：

- 标题图不是解释全文，而是在 3 秒内让读者判断“这条和我有没有关系”。
- 每张标题图必须有一个主视觉锚点：超大标题、巨型数字、手机/截图、结果卡片、人物或改前改后对比。
- 先选模板，再生成图片：冲突型、数字型、截图型、情绪型、生活方式拼贴型。
- X 标题图优先 5:2：左侧 45%-60% 放主标题和副标题，右侧 40%-55% 放视觉锚点。
- 主标题占 60%-75% 文字视觉重量，副标题占 20%-30%，标签/角标占 5%-10%。
- 主标题优先 2-8 个汉字；超过 12 个字就拆成主标题 + 副标题。
- 图片最多 2-3 个文字区，不塞正文段落。
- 缩小到 80px 宽时，主标题和主锚点仍要可读。

模板映射：

- 反直觉观点 / 错误纠正 / AI 工具判断：优先冲突型。
- 步骤教程 / 方法清单 / 工具技巧：优先数字型。
- 工具测评 / 工作流案例 / 结果展示：优先截图型。
- 个人经历 / 情绪故事：少量使用情绪型。
- AI 自媒体创作和生活方式副线：可用涂鸦描边或阳光胶片拼贴。

封面生成前内部先确定：

```text
模板类型：
一眼承诺：
视觉锚点：
锚点位置：
文字层级：
```

封面诊断要看：

- 模板是否匹配内容。
- 一眼承诺是否清楚。
- 是否有唯一主锚点。
- 主标题是否过长。
- 对比度是否足够。
- 是否通过 80px 缩略图测试。
- 是否有可信信号，如截图、工具界面、结果卡片、真实案例。
- 是否避免伪造 Logo、收益、二维码、私密聊天或平台数据。

## 禁用或少用

少用：

- “震撼”
- “炸裂”
- “保姆级”
- “无脑”
- “稳赚”
- “颠覆”
- “全网最全”

避免：

- 复制原文结构再换词。
- 把别人的截图直接当配图。
- 没验证就写成确定事实。
- 对收益做承诺。
- 引导刷量、规避平台风控。

## 每篇文章发布前检查

1. 第一条有没有清晰 Hook？
2. 这篇有没有一个自己的判断？
3. 有没有把信息重组成自己的结构？
4. 每条是否只讲一个点？
5. 有没有 TL;DR？
6. CTA 是不是具体问题？
7. 配图能否一眼看清？
8. 有没有来源说明？
9. 有没有夸大收益或官方身份风险？
10. 发布后是否记录数据和反馈？
11. 这篇是否有“实测、对比、工作流、作品”里的至少一个强标签？
12. 是否留下了一个可以引导评论的问题？

## 发布后复盘字段

每篇文章发布后，建议记录：

```text
发布时间：
标题：
Hook：
配图：
24h 展示：
24h 互动：
评论里最多的问题：
哪句话被转发/引用最多：
下次保留：
下次删掉：
```

风格不是一次定死的，而是靠每篇文章的反馈慢慢收敛。

## 固定交付物

以后每次写 X 文章，默认都要配套生成对应视觉素材：

1. 标题图片：优先使用 X 推荐的 5:2 比例，文字要少、要大、手机缩略图一眼能看清。
2. 文章配图：根据文章结构至少生成 1-3 张正文插图，用来解释核心流程、对比关系或关键结论。
3. 配图说明：在文章目录里保留 `image-guide.md`，说明每张图的用途和推荐插入位置。
4. 正文位置标记：在 `article.md` 对应段落前后标出图片插入位置，格式优先用 `配图位置：插入 assets/xx.png`，不要只写在 `image-guide.md`。
5. 场景化画面：不要所有图都是文字和卡片。至少一张图要结合文章内容加入人物、物体、设备、桌面、手机、代码界面、工具界面、数据面板、工作现场或生活场景。
6. 生成脚本：把可复用的图片生成脚本放到 `tools/`，图片输出到 `assets/`。
7. README 索引：文章、配图、说明和脚本都要登记到根目录 `README.md`。

配图不是装饰，而是文章的一部分：标题图负责抓注意力，正文图负责降低理解成本。

### 视觉风格不要单一化

已学习多风格封面参考图：`references/visual-style/multi-cover-style-board-reference.png`。

对应规则笔记：`references/visual-style/multi-cover-style-board-notes.md`。

后续不要所有标题图都使用同一种红黑白高冲击模板。生成图片前先按内容类型选择视觉方向：

- 工具测评：截图测评 / 数据面板 / 霓虹科技。
- 方法清单：巨型数字 / 纸感清单。
- 封面拆解：改前 VS 改后 / 粗字冲击。
- 账号风险：冲突人像 / 粗字冲击。
- 个人故事：情绪人像 / 夜间生活方式。
- 治愈、学习、长期主义：治愈极简。

连续 3 篇文章标题图尽量不重复同一模板。

同时避免“全是文字”的图。标题图可以用大字抓注意力，但画面里最好有一个和主题相关的场景锚点：

- AI 编程：代码窗口、终端、开发者桌面、浏览器预览、任务面板。
- AI 自媒体：手机、相机、时间线、封面墙、内容日历。
- AI 写歌：耳机、波形、音乐工作台、麦克风、播放界面。
- 自动化：机器人通知、流程节点、服务器面板、工厂设备、仪表盘。
- 账号增长：评论卡片、数据面板、内容看板、发布日历。
- 个人复盘：咖啡馆、书桌、笔记本、人像、夜间工作场景。

## X 增长工作流规则

这套规则来自对 X 内容增长方法的吸收，但不把“算法权重”写成固定公式。以后写文章时，优先围绕“关注理由”设计内容，而不是只追求信息密度。

### 1. 核心目标

每篇文章都要回答一个问题：

> 读者看完以后，为什么要关注这个账号，而不是只收藏这条内容？

涨粉最快的内容不是“信息最多”，而是让陌生人快速判断：

- 这个账号持续值得看。
- 这个人有稳定领域。
- 这个人不只是转述工具，而是能实测、拆流程、做成作品。

### 2. 内容类型优先级

以后 AI 相关选题优先落到 6 类内容里：

1. 反直觉观点：给出一个明确判断，让人停下来。
2. 可保存方法：步骤、清单、流程、模板，让人收藏。
3. 实测对比：同一任务、同一 Prompt、不同工具，给出结论。
4. 完整工作流：从输入到输出，把工具串成可复用流程。
5. 真实案例复盘：展示做出来的结果、踩坑和改法。
6. 高质量战略回复：在别人的讨论里提供增量观点。

如果一篇文章不属于以上任意一类，发布前要重新判断它是否偏离账号主线。

### 3. 选题过滤

每个选题发布前，至少满足下面 2 条：

- 有明确受众：AI 编程、AI 创作、自媒体自动化、工具实测用户。
- 有一个反常识判断或具体问题。
- 有可执行步骤，而不是只有观点。
- 有实测、截图、流程、文件或案例作为证明。
- 能自然引出一个具体评论问题。
- 能配一张标题图和至少一张正文解释图。

避免随机追热点。热点只有在能接回“AI 工具实测 / AI 工作流 / 自动化落地 / 想法变作品”主线时才写。

起草前先给文章归类：

```text
AI 编程 / 自动化
AI 自媒体创作
AI 编程 + 自媒体自动化结合
工具对比
工作流拆解
案例复盘
方法清单
反直觉观点
战略回复延展
```

如果归不了类，说明选题还不够具体，先不要写成长文。

账号内容比例默认按 70/30 执行：70% AI 编程与自动化，30% AI 自媒体创作。能把两者结合成“可运行工具 / 自动化流程 / 内容生产系统”的选题，优先级最高。

### 4. Thread 结构

优先写 5-12 条 Thread。结构固定为：

```text
Hook -> 问题/误区 -> 框架 -> 具体步骤/案例 -> 踩坑 -> 总结 -> 具体 CTA
```

第 1 条只做一件事：让读者停下来。

中间每条只讲一个点，尽量配截图、流程图或对比图。

最后不要只写“关注我”。优先问一个具体问题，例如：

- 你最想自动化哪个重复工作？
- 你更想看工具对比，还是完整工作流？
- 这个流程里你最担心哪一步？

### 5. 战略回复

小号阶段，回复是内容分发的一部分。

每天预留 20-40 分钟，回复同领域里比自己大一些的账号。回复对象优先：

- AI 工具测评账号
- AI Coding / Cursor / Claude / Codex 相关账号
- AI 创作、自媒体、自动化、No-code 账号
- 有真实讨论而不是纯情绪流量的帖子

有效回复必须提供增量，至少符合一条：

- 补一个实测细节
- 给一个不同场景
- 提一个风险边界
- 补一个参数、流程或失败案例
- 问一个能继续讨论的具体问题

禁用回复：

- “学习了”
- “太强了”
- “收藏了”
- 无关蹭热门
- 批量复制同一句
- 互赞互粉、诱导刷互动

### 6. 发布节奏

0-1000 粉阶段，优先稳定形成识别度：

- 每天 1 条高质量短帖。
- 每周 2 条结构化 Thread。
- 每周至少 1 条实测或案例复盘。
- 每天 20-40 分钟战略回复。
- 每周做一次数据复盘。

不要为了数量牺牲定位。账号早期最重要的是让读者连续 3 次看到你时，知道你在解决什么问题。

### 7. 数据复盘

发布后不要只看浏览量。优先记录：

- 这条是什么类型：观点 / 方法 / 实测 / 工作流 / 案例 / 回复延展。
- Hook 是什么。
- 配图是什么。
- 是否带来个人资料访问。
- 是否带来关注。
- 是否有高质量评论。
- 是否被引用或转发。
- 评论区暴露了什么新选题。

浏览量是噪音，关注理由才是资产。复盘时重点看哪类内容更能让人关注，而不是哪类内容偶然曝光更高。

### 8. 合规边界

不要把增长策略写成平台操纵技巧。

避免：

- 重复、无关、高频回复
- 堆 hashtag
- 互赞互粉
- 诱导刷评论
- 搬运未验证内容
- 把未经证实的算法说法写成确定事实

可持续的增长来自真实内容和真实互动，不来自规避平台规则。
