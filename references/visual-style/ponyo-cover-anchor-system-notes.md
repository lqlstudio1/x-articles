## 核心吸收

这个 skill 的核心不是某一种颜色或某一套模板，而是：

```text
封面点击力 = 信息密度 × 视觉锚点
```

- 信息密度：读者在缩略图里能不能立刻看懂“这条和我有什么关系”。
- 视觉锚点：画面里有没有一个足够强的位置，让眼睛先停住。

后续给 X 文章做标题图时，不能只做“漂亮背景 + 标题”，必须先回答：

1. 这张图的承诺是什么？
2. 视觉锚点在哪里？
3. 缩小到手机信息流里，主标题还能不能看清？

## 模板选择

### 1. 冲突型

适合：

- 反直觉观点
- 错误纠正
- AI 工具判断
- 平台机制批判

X 文章适配：

- 左侧放超大冲突标题。
- 右侧放人物、卡片、工具界面或对立符号。
- 红黑白、高对比、斜切色块、笔刷线适合这类。

### 2. 数字型

适合：

- 步骤教程
- 清单
- 方法论
- 工具技巧

X 文章适配：

- 数字本身就是锚点。
- 标题短，数字大。
- 适合“3 个锚点 / 5 步流程 / 7 个错误”。

### 3. 截图型

适合：

- 工具测评
- 工作流案例
- 结果展示
- 改前改后

X 文章适配：

- 右侧放手机框、界面卡片、数据面板或成果截图。
- 叠加红色手写圈、箭头、下划线。
- 不要伪造真实平台数据、收益、Logo 或私密截图。

### 4. 情绪型

适合：

- 个人经历
- 情绪故事
- 生活方式
- 自媒体创作副线

X 文章适配：

- 少量使用。
- 账号主线偏 AI 编程和自动化，除非是故事型 Thread，否则不优先。

### 5. 涂鸦描边 / 阳光胶片拼贴

适合：

- 旅行、家居、Vlog、Citywalk、生活方式。

X 文章适配：

- 可用于 AI 自媒体创作、副线内容和封面拆解。
- 不作为 AI 编程 / Codex / 自动化主线默认风格。

## X 5:2 标题图适配规则

原 skill 默认偏小红书 3:4。当前工作区主要发 X 文章，标题图优先用 5:2，所以适配为：

```text
左侧 45%-60%：主标题 + 副标题 + 标签
右侧 40%-55%：视觉锚点，如卡片、手机、截图、数字、人物、结果预览
底部或角落：Prompt / Skill / 模板 / 工作流 等小标签
```

文字层级：

```text
主标题：60%-75% 视觉重量
副标题：20%-30% 视觉重量
标签/角标：5%-10% 视觉重量
```

缩略图规则：

- 主标题缩到 80px 宽仍要能辨认。
- 全图最多 2-3 个文字区。
- 不把正文段落塞进图里。
- 主标题优先 2-8 个汉字；超过 12 个字就拆成标题 + 副标题。

## 标题图生成前的固定输出

以后做标题图前，先内部确定这 5 项：

```text
模板类型：
一眼承诺：
视觉锚点：
锚点位置：
文字层级：
```

如果需要对外写入 `image-guide.md`，则补充：

```text
配色：
建议插入位置：
缩略图检查：
```

## 封面诊断清单

旧图或生成图要用下面 10 项检查：

1. 模板是否匹配内容。
2. 一眼能不能看懂利益点。
3. 有没有唯一主锚点。
4. 锚点位置是否在左上、上方中心、中心或右侧主视觉区。
5. 80px 缩略图是否还能读主标题。
6. 主标题、副标题、标签是否有清晰层级。
7. 主标题和背景对比是否足够。
8. 主标题是否过长。
9. 是否有可信信号，如截图、数字、结果、工具界面、真实案例。
10. 是否符合平台比例和阅读习惯。

## 禁区

- 不复制仓库示例图片和具体素材。
- 不硬塞 `波妞`、`PONYO`、作者 Logo 或任何水印。
- 不伪造平台 Logo、真实数据、收益截图、二维码或私密聊天。
- 不把所有图都做成高噪音封面；正文解释图仍以清楚为第一优先级。
- 不为了点击牺牲文章边界，例如收益承诺、夸张保证或诱导刷量。
