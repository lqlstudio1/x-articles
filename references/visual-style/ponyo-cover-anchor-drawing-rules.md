## 核心吸收

封面和标题图的目标不是“好看”，而是让用户在信息流里 3 秒内判断：这条内容和我有没有关系。

默认公式：

```text
封面点击力 = 信息密度 × 视觉锚点
```

- 信息密度：缩略图里能不能一眼看懂利益点。
- 视觉锚点：画面里有没有一个明确位置让眼睛停住。

## 默认行为

后续绘制 X 标题图、小红书封面、公众号头图、文章配图时，除非用户明确说“只要背景”，默认生成完整成品图：

- 有主标题；
- 有副标题或底部标签；
- 有主视觉锚点；
- 有色块、卡片、截图框、数字、人物或结果面板；
- 有清晰文字层级；
- 不生成空背景等用户后期叠字。

## 模板选择

### 冲突型

适合：

- 反直觉观点；
- 错误纠正；
- AI 工具判断；
- 平台机制批判；
- 方法论重构。

常用结构：

```text
短而狠的主标题 × 红黑白强对比 × 卡片/人物/对立符号
```

X 5:2 适配：

- 左侧放超大标题；
- 右侧放结果卡片、AI 答案卡、警示牌、工具面板；
- 主标题控制在 2-8 个汉字，长标题拆成主标题 + 副标题。

### 数字型

适合：

- 步骤教程；
- 方法清单；
- 技巧总结；
- 工具榜单；
- “5 步 / 7 个 / 10 条”类内容。

常用结构：

```text
巨型数字 × 色块分割 × 简短承诺
```

规则：

- 数字本身就是锚点；
- 数字高度可占画面 30%-40%；
- 数字旁不要塞长句。

### 截图型

适合：

- 工具测评；
- 工作流案例；
- 结果展示；
- 改前改后；
- 数据面板。

常用结构：

```text
截图 / 手机框 / 面板 × 红色手写标注 × 低饱和底色
```

规则：

- 真实感比装饰更重要；
- 可以用通用 UI mockup，但不能伪造真实平台数据、收益截图、Logo 或私密聊天；
- 标注要像手写，不能像规整图表。

### 情绪型

适合：

- 个人经历；
- 生活方式；
- 情绪故事；
- 自媒体副线内容。

账号主线偏 AI 编程和自动化，所以情绪型少量使用，除非文章本身是故事或个人复盘。

### 涂鸦描边小清新

适合：

- 旅行；
- 家居；
- Vlog；
- Citywalk；
- 咖啡探店；
- 生活方式教程。

规则：

- 真实照片质感；
- 一个主体加粗白描边；
- 手帐贴纸式标题；
- 涂鸦元素要少，只负责引导视线。

### 阳光胶片拼贴白描边

适合：

- 旅行 / Citywalk；
- 穿搭；
- 护肤；
- 居家运动；
- 生活方式内容。

规则：

- 顶部 18%-25% 放胶片条或拼贴小图；
- 主体在右侧或右下，占画面 42%-58%；
- 主体用不规则白描边抠出；
- 左侧放手写刷字标题；
- 黄油黄、鼠尾草绿作为关键词强调；
- 只保留一个主标题、一个副标题，最多一个小注释。

## X 文章标题图适配

原 skill 偏小红书 3:4，本工作区默认做 X 文章标题图，优先 5:2。

默认布局：

```text
左侧 45%-60%：主标题 + 副标题 + 底部标签
右侧 40%-55%：视觉锚点，如卡片、手机、截图、数字、结果面板
背景：高对比色块、纸感纹理、红色笔刷或科技暗底
```

文字层级：

```text
主标题：60%-75% 文字视觉重量
副标题：20%-30%
标签 / 角标：5%-10%
```

图片文字区最多 2-3 个，不塞正文段落。

## 诊断规则

旧图或生成图要按 10 点检查：

1. 模板是否匹配内容。
2. 一眼承诺是否清楚。
3. 是否只有一个主视觉锚点。
4. 锚点位置是否在左上、上方中心、中心或右侧主视觉区。
5. 80px 宽缩略图里主标题是否仍可读。
6. 主标题、副标题、标签是否有清晰层级。
7. 主标题和背景对比度是否足够。
8. 主标题是否过长。
9. 是否有可信信号，如截图、结果卡片、真实案例、对比关系。
10. 是否符合平台比例和阅读习惯。

评分时使用：

```text
信息密度：1-10
视觉锚点：1-10
缩略图可读性：1-10
色彩对比：1-10
信任感 / 真实感：1-10
```

## 生成提示词硬约束

无论使用图片模型还是本地脚本生成，都要遵守：

- 中文文字必须大、清晰、正确；
- 主标题优先短句，不用完整文章标题；
- 不生成乱码、随机英文、随机品牌、随机 Logo；
- 不添加 `波妞`、`PONYO`、作者水印、`@` 账号，除非用户明确提供自己的账号标识；
- 不伪造收益、浏览量、平台排名、二维码、私密聊天、真实后台；
- 不复制该 GitHub 仓库示例图或具体素材，只使用方法论和布局规则；
- 正文解释图可以更清晰克制，不必全部做成高噪音封面。

## 本工作区默认风格合并

对于 @superfang119 的 AI / Codex / 自动化内容，默认优先：

- 红黑白高冲击封面；
- 5:2 横向 X 标题图；
- 左侧超大中文标题；
- 右侧卡片 / 手机框 / 结果面板 / 流程节点；
