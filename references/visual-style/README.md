## 适用场景

这种风格优先用于标题图、封面图、强 Hook 图，尤其适合：

- 方法论文章
- 工具 Skill / 工作流主题
- 反直觉观点
- 小红书 / X / 自媒体封面分析
- “一眼看懂核心卖点”的教程型内容

正文解释图不必完全照搬这种高强度风格，可以只借用红黑白高对比、卡片透视和醒目标签。

## 视觉特征

- 白色或浅灰纸感底，不是纯深色底。
- 红、黑、白作为主色，红色负责攻击性和点击感，黑色负责标题重量。
- 超大中文主标题，占据画面左侧 45%-60%。
- 字体要厚重、粗黑、带轻微笔刷或破损质感。
- 关键词用红黑交替强调，不要整句同色。
- 用红色笔刷、喷溅、下划线、角标、准星、星芒强化冲击。
- 右侧可以放立体卡片、手机封面、网页截图、工具界面或成果预览。
- 可以用鼠标指针、靶心、箭头、爱心、定位点等符号强化“点击/命中/增长”隐喻。
- 信息层级必须明确：主标题最大，副标题其次，底部标签最小。

## 构图模板

优先采用横向 5:2：

```text
左侧：巨大标题 + 副标题 + 底部标签
右侧：立体卡片 / 手机界面 / 成果预览
背景：红色笔刷和浅色纸感纹理
点缀：准星、箭头、鼠标、星芒、下划线
```

## 标题规则

- 主标题控制在 2-6 个汉字，例如“视觉锚点”“标题钩子”“爆款封面”。
- 副标题控制在 10-18 个字，突出结果或判断。
- 底部标签可以写“附 Prompt / Skill / 模板 / 工作流”。
- 图片里不要塞完整段落。

## 颜色建议

```text
背景：#F7F4EF / #FFFFFF
主红：#E60012 / #F20D23
主黑：#050505
辅助灰：#E9E5DF
阴影：rgba(0,0,0,0.18)
```

## 使用边界

- 参考构图和视觉语言，不复制原图内容、版式细节或具体素材。
- 不要在所有文章里都使用这种高噪音封面；工具实测、方法论、强观点优先使用。
- 如果主题偏 Codex / AI 编程 / 自动化，也可以保留原来的深色科技风；此风格作为“高点击封面”备选。
- 生成图片时优先确保手机缩略图里主标题可读。
