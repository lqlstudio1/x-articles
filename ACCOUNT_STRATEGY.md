## 内容优先级

### 1. AI 工具实测对比 Thread

优先级：最高。

适合内容：

- Seedance 2.0 mini vs 4k
- Codex vs Claude Code
- 同一 Prompt 在不同工具里的结果差异
- 成本、速度、质量、适用场景对比

标准结构：

```text
Hook：相同 Prompt 下，A 和 B 到底差多少？
1. 测试目标
2. 相同 Prompt
3. A 的结果
4. B 的结果
5. 成本 / 时间 / 质量对比
6. 我的结论：什么时候用 A，什么时候用 B
CTA：你还想看哪个工具对比？
```

### 2. 完整工作流 Thread

优先级：最高。

适合内容：

- Codex 读项目 -> 计划 -> 修改 -> 测试 -> Review
- Claude Code + Codex 联动
- AI 视频生成 -> 剪辑 -> 发布
- 图文教程生产流水线
- 浏览器自动化 / 后台填表 / 批量处理

标准结构：

```text
Hook：很多人只会用 AI 单点生成，但真正值钱的是完整工作流。
1. 最终成果
2. 输入材料
3. 工具组合
4. 每一步怎么跑
5. 中间踩坑
6. 可复用模板
7. 下次怎么优化
CTA：评论区说一个你想自动化的流程。
```

### 3. Prompt + 案例拆解

优先级：高。

适合内容：

- 贴 Prompt
- 展示生成结果
- 拆解为什么这么写
- 给出可复制版本

注意：

Prompt 不要孤立发。必须配最终结果、失败版本或改进逻辑。

### 4. 想法 -> 可运行作品

优先级：高。

这是最符合 Bio 的长期品牌资产。

适合标题：

- 我用 AI 把一个想法做成了可运行网页
- 一个周末，用 Codex 做完一个小工具
- 从一句需求到上线页面，我让 AI 跑完整链路

## 战略回复

回复模板：

```text

```

```text

```


## 增长内容矩阵

以后账号内容不要只按“发什么工具”来排，而要按“给读者什么关注理由”来排。

### 6 类固定栏目

1. 观点帖：建立判断
   - 例：AI 写作最大的问题不是 AI 味，而是没有判断。
   - 目标：让陌生人知道这个账号有立场。

2. 方法帖：提高收藏
   - 例：我做一篇 X 文章的流程：选题 -> 正文 -> 短版 -> 配图 -> 复盘。
   - 目标：让读者觉得下次还要来看。

3. 实测帖：建立信任
   - 例：同一个任务，Codex 和 Claude Code 到底差在哪里。
   - 目标：证明不是资讯搬运，而是真的跑过。

4. 工作流帖：强化主线
   - 例：把单点 AI 工具串成可复用生产链路。
   - 目标：匹配 Bio 里的“把想法变成能跑的作品”。

5. 案例帖：沉淀资产
   - 例：从一句需求到一个可运行页面，完整复盘。
   - 目标：让账号有作品，而不是只有观点。

6. 战略回复：借力曝光
   - 例：在 AI Coding / 自动化 / AI 创作大号评论区补充实测细节。
   - 目标：用高质量回复让精准用户点进主页。

## 双主线内容规则：AI 编程 + AI 自媒体

账号当前最适合走双主线，但比例要清楚。

推荐内容比例：

```text
70% AI 编程 / 自动化
30% AI 自媒体创作
```

原因：

- AI 编程和自动化更容易建立专业信任。
- AI 自媒体创作更容易扩大受众和互动。
- 两者结合最有辨识度：用 AI 编程做自媒体自动化工具。

以后写选题时，优先判断它属于哪一类：

1. AI 编程 / 自动化主线
2. AI 自媒体创作副线
3. 两者结合的高价值主线

如果一篇内容既能体现“会写代码”，又能体现“会做内容”，优先级最高。
