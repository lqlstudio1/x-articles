# 项目地图

## 根目录

- `README.md`：文章索引，按时间和专题整理。
- `WRITING_STYLE.md`：写作风格、文章结构、视觉风格、增长工作流规则。
- `ACCOUNT_STRATEGY.md`：@superfang119 账号定位、内容矩阵、双主线策略、Thread 模板。
- `AGENTS.md`：长期协作规则和禁区。
- `progress.md`：阶段进度和验证记录。
- `daily_task.md`：当前任务、优先级、阻塞项和验收标准。
- `mistakes.md`：错误、原因、修复和复测记录。
- `references/visual-style/`：长期视觉参考图、封面风格说明和 `ponyo-cover-anchor-system` 学习笔记。
  - `ponyo-cover-anchor-drawing-rules.md`：将 GitHub 仓库 `ponyodong2026/ponyo-cover-anchor-system` 转为本工作区绘图执行规则。
  - `multi-cover-style-board-reference.png` / `multi-cover-style-board-notes.md`：多风格封面参考板，用于避免标题图风格单一化。

## 文章目录规则

文章按日期和专题放置：

```text
YYYY-MM-DD/topic-slug/
```

常规结构：

```text
article.md
article-short.md
article-medium.md
assets/
image-guide.md
tools/
source-notes.md
```

不是每篇都必须有 `article-medium.md` 或 `source-notes.md`，但凡涉及资料边界、平台规则、外部事实，优先保留 `source-notes.md`。
