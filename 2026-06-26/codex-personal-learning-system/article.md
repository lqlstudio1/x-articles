# Codex 个人学习系统

## 原文分析

原文《AI 时代，如何用 Codex 打造个人学习系统》的核心价值：

- 把“AI 辅助学习”从问答升级成长期系统。
- 把学习过程设计成本地仓库，而不是一份一次性计划。
- 用 `AGENTS.md` 写运行规则，用文件保存目标、进度、知识地图、任务、练习、错误和项目成果。
- 强调 Codex 的角色不是替你学习，而是维护状态、推进任务、记录错误、运行检查。
- 最重要的判断是：AI 时代缺的不是答案，而是学习闭环。

原文可优化的地方：

- 篇幅很长，适合长文，但 X Thread 需要更强 Hook 和更短路径。
- 模块很多，容易读完觉得“很完整”，但不知道第一步做什么。
- 对 @superfang119 来说，最适合改成“工作流案例”：把学习当成一个可以运行、检查、复盘的项目。

改写角度：

> AI 学习最大的坑，是把“拿到答案”误认为“获得能力”。用 Codex 做学习系统，本质是把学习当项目来跑。

## 官方事实校准

依据 OpenAI Codex 官方手册：

- Codex 是 OpenAI 的 coding agent，可以写代码、理解代码库、Review、Debug、自动化开发任务。
- 官方最佳实践建议把 Codex 当成可持续配置和改进的队友，而不是一次性助手。
- 提示任务时最好给目标、上下文、约束、完成标准。
- `AGENTS.md` 适合放仓库里的长期工作规则。
- Automations 可以执行周期性后台任务，但要注意本机运行状态、沙盒和权限风险。

## X 发布版：10 条 Thread

1/

[配图：assets/01-title-5x2.png]

很多人用 AI 学习，其实只停留在第一层：

问问题，拿答案。

这当然有用。

但问题是，答案来得太快，你很容易产生一种错觉：

“我好像已经学会了。”

真正到项目里一做，才发现脑子里只有零散概念，没有可调用的方法。

2/

我看完一篇用 Codex 打造个人学习系统的长文后，最认同一个判断：

AI 时代真正稀缺的，不是知识。

而是学习闭环。

你要知道：

今天学什么。
怎么练。
怎么验收。
哪里错了。
什么时候复测。
最后如何进入一个真实作品。

3/

普通 AI 对话很适合解释概念。

但它不擅长长期管理学习状态。

比如：

你昨天学到哪里？
最近三次都错在哪里？
哪个概念需要三天后复测？
最终项目推进到哪一步？

这些信息如果只放在聊天记录里，很快就散了。

4/

[配图：assets/02-learning-loop.png]

所以我更喜欢这个思路：

把学习变成一个本地仓库。

里面不是只放笔记，而是放：

- 目标
- 知识地图
- 每日任务
- 练习记录
- 错误与复测
- 项目进度
- 最终产出

学习开始变成一个可以运行的项目。

5/

[配图：assets/03-minimal-repo.png]

最小版本其实不用复杂。

先建 5 个文件就够：

```text
AGENTS.md        # 学习规则
progress.md      # 当前进度
knowledge_map.md # 知识地图
daily_task.md    # 今日任务
mistakes.md      # 错误和复测
```

这 5 个文件解决一个核心问题：

不用每次重新解释背景。

6/

Codex 在这里的价值，不是“比 ChatGPT 更会讲课”。

而是它能读文件、改文件、理解目录结构、运行检查，并基于已有状态继续推进。

你可以让它每天先读 `progress.md` 和 `daily_task.md`。

然后只做一件事：

找到第一个未完成任务，继续推进。

7/

我会给 Codex 这样的规则：

不要一次性把答案全部给我。

按这个节奏来：

解释 -> 练习 -> 检查 -> 记录。

我先回答，你再纠错。

错了就写进 `mistakes.md`，并安排复测。

这样 AI 不是替我学习，而是在提高我的思考密度。

8/

学习系统最容易缺的，不是计划。

而是反馈。

错误不要只写“这里错了”。

至少要记录：

- 错误是什么
- 当时怎么理解的
- 正确理解是什么
- 根因是什么
- 下一次怎么复测

没有复测的错题本，只是失败收藏夹。

9/

最后一定要有项目。

如果你学 Agent，不要目标写成“掌握 Agent”。

更好的目标是：

30 天内做一个能运行的个人任务拆解 Agent。

每天学到的概念，都要变成项目里的一个组件：

提示词、流程图、测试用例、工具定义、失败回退、使用文档。

10/

TL;DR：

- AI 学习不要只问答案
- 把学习做成本地仓库
- 用 `AGENTS.md` 写学习规则
- 用 `progress.md` 维护状态
- 用 `mistakes.md` 做复测
- 用项目证明自己真的会了

Codex 最适合做的，不是替你学。

而是把学习变成能持续运行、检查和迭代的系统。

你现在最想用 AI 学什么？评论区说一个主题，我可以按这个结构拆一个最小学习仓库。
